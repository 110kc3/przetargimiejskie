# Spike — Chojnice (Pomorskie · powiat chojnicki)

> **Status:** spike LIVE-VERIFIED — 2026-06-28. VERDICT: BUILD (Medium effort).

## TL;DR

Gmina Miejska Chojnice (pop. ~40k) actively sells municipal flats via *przetarg ustny nieograniczony* — confirmed from live page fetches. The city hall website (miastochojnice.pl) publishes both announcement and result notices as WordPress posts, with the actual auction document attached as a small PDF (~100 KB). Achieved-price notices are published as companion PDF attachments on a separate results board. Volume is low-to-moderate (~1–3 flat auctions per year visible in recent history), but the data is cleanly structured and bot-friendly. ZGM Chojnice (housing manager) exists but flat auction announcements are published directly by the Urząd Miejski, not ZGM. BIP mirror at bip.miastochojnice.pl is also present (category 616) but the main UM site is the canonical live board.

## 1. Sells municipal property at auction?

**YES — confirmed LIVE.** The city sells *lokale mieszkalne* via *przetarg ustny nieograniczony* (unlimited oral auction). Recent confirmed examples:

- **31.03.2026** — *przetarg ustny nieograniczony na sprzedaż lokalu mieszkalnego przy Placu Jagiellońskim 6/5* — announcement published 2026-02-26, PDF attachment [104 KB].
- **21.10.2025** — *wynik przetargu na lokal mieszkalny* — result notice published (PDF).
- Earlier in 2025 (search snippet): auction for lokal mieszkalny on Osiedle Bajkowe and ul. Kościerska area.
- Historical: *przetarg ustny nieograniczony na zbycie lokalu mieszkalnego — 5.10.2021*.

This is **NOT** bezprzetargowy — these are open competitive auctions, not preferential sales to tenants. The flats are municipally owned and disposed through standard auction procedure.

## 2. Where published? (hosts + boards, URLs)

**Primary board — announcements:**
- `https://miastochojnice.pl/kat/przetargi-na-nieruchomosci-miejskie/` — WordPress category listing, paginated (13 pages of history confirmed). Contains all property auctions including flats, land, commercial.

**Primary board — results (achieved price):**
- `https://miastochojnice.pl/kat/przetargi-na-nieruchomosci-miejskie-wyniki/` — companion results board (7 pages of history). Each result post links to a PDF with the achieved price.

**BIP mirror:**
- `https://bip.miastochojnice.pl/616.html` — BIP "Przetargi na nieruchomości miejskie" — parallel publication but main UM site appears to be the canonical live board. BIP page was very large (68k chars) suggesting it is a listing aggregator.
- `https://bip.miastochojnice.pl/615.html` — "Nieruchomości miejskie przeznaczone do zbycia"

**ZGM Chojnice** (Zakład Gospodarki Mieszkaniowej Sp. z o.o., zgmchojnice.pl): the housing manager exists but their przetargi board at `https://zgmchojnice.pl/przetargi` covers building maintenance contracts and bare land sales — NOT the flat sale auctions. Flat auctions flow through UM directly.

No aggregator (e.g., przetargi.egospodarka.pl) appears to carry the Chojnice flat listings.

## 3. Format + rendering

- **Listing pages:** WordPress 6.9.4 HTML, standard server-rendered pagination. No JS-gated content, no SPA. Clean `<a>` links in the main content area. No auth required.
- **Individual auction posts:** WordPress post with a short header (title = auction date + description) and a single "Dokumenty do pobrania" section linking to a PDF attachment.
- **PDF attachments:** Small PDFs (~100–110 KB). These are the actual auction notices with all legal details (description of property, starting price, wadium, auction date/location). Format is text-PDF (generated by city staff from a Word template) — likely machine-readable, but OCR may be needed for older entries.
- **Result posts:** Same WordPress structure — post title states the result, PDF attachment contains the achieved-price protocol.
- **No bot-blocking observed** — Cloudflare CDN headers present but no CAPTCHA or rate limiting encountered during fetch. Standard `robots.txt` is expected (not checked).
- **RSS feed** likely available via WordPress (`/feed/` endpoint on category) — not verified but standard WordPress behavior.

## 4. Volume + achieved-price stream

- **Announcement board:** 13 pages observed (10 items/page = ~130 total entries, spanning back to at least 2021). Mix of flats, land, commercial properties.
- **Flat-specific auctions:** Approximately 2–4 per year based on visible entries (2025: at least 2; 2026: 1 confirmed so far). Low but consistent volume.
- **Results board:** 7 pages (~70 result notices). Each flat auction generates a matching result post with PDF containing achieved price.
- **Achieved-price stream:** YES — available as separate board + PDF. The result for Pl. Jagielloński 6/5 (31.03.2026) was published on 07.04.2026 (7-day lag). Clean pairing: announcement post → result post, same slug pattern (`przetarg-...` vs `wyniki-przetargu-...`).

## 5. Adapter effort + verdict

**Closest analog:** Tarnowskie Góry or Bytom pattern — generic city-BIP WordPress board with PDF attachments for auction details. NOT like Gliwice (which has a richer HTML inline format).

**Adapter design:**
1. Scrape paginated category listing at `miastochojnice.pl/kat/przetargi-na-nieruchomosci-miejskie/` — detect posts with "lokal mieszkalny" or "lokalu mieszkalnego" in the title.
2. For each matching post, download the PDF attachment and extract: address, area, cena wywoławcza, wadium, auction date.
3. Scrape the results board (`/kat/przetargi-na-nieruchomosci-miejskie-wyniki/`) for matching result PDF — extract achieved price.
4. PDF extraction: text-PDF expected (not scanned), so pdfplumber/PyMuPDF should work without OCR.

**Blockers:**
- The authoritative data (price, area, legal description) is inside the PDF, not in the HTML post body. HTML posts are stub-only — just a title and a file link.
- PDFs must be downloaded and parsed per announcement. At ~100 KB each this is trivial.
- No structured metadata API or JSON feed confirmed.

**Risks:**
- Low volume (~2–4 flat auctions/year) — marginal value per adapter, but the data is clean.
- PDF template may vary across years — needs field-extraction testing on a few samples.
- BIP mirror and UM site may diverge (monitor both or default to UM site).

**Effort:** Medium — WordPress list scraper (1 day) + PDF field extractor (1–2 days, dependent on PDF template consistency) + result matcher (0.5 day). Total ~2.5–3 days.

**VERDICT: BUILD** — confirmed flat auctions, both announcement and result/achieved-price published, WordPress HTML fully crawlable, no bot blocks, PDF is text-format. Low volume is the only downside. Confidence: High.
